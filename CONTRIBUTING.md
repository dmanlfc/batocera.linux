# Contributing to Batocera Linux

## Thank You

Thank you for your interest in contributing to Batocera Linux.

Every contribution helps improve the project for users around the world.

---

## Contribution Philosophy

The preferred development model of Batocera Linux is:

> Improve upstream so everyone benefits.

Contributions shared with the project help reduce duplication of effort and strengthen the ecosystem as a whole.

---

## Ways to Contribute

Contributions may include:

- Code
- Bug reports
- Documentation
- Testing
- Localization
- Artwork
- Packaging
- Hardware validation
- Community support

---

## Pull Requests

Contributors should:

- Follow project coding standards
- Provide clear commit messages
- Document significant changes
- Explain the purpose of submissions

---

## Collaboration

Contributors are encouraged to:

- Discuss significant changes before implementation
- Work collaboratively with maintainers
- Seek consensus where appropriate

---

## Forks

Forks are permitted under GPLv3.

However, contributors are encouraged to submit improvements upstream whenever practical so that the broader Batocera Linux community can benefit.

---

## Respect

All contributors are expected to treat one another respectfully and professionally.

Constructive discussion and good-faith collaboration are essential to the success of the project.
