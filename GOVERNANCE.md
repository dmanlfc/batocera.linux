# Batocera Linux Project Governance

## Purpose

Batocera Linux is a community-driven open-source operating system built through the collective efforts of developers, maintainers, testers, translators, artists, documentation writers, and community members.

This document defines the guiding principles, values, and governance philosophy of the Batocera Linux Project.

## Project Philosophy

Batocera Linux is more than a software distribution—it is a collaborative community project.

Every release represents thousands of hours of voluntary work contributed by individuals who share a common goal: creating an accessible, high-quality, and sustainable open-source gaming platform.

The Batocera Linux Project believes that open-source software is strongest when improvements are shared openly and contributed upstream for the benefit of everyone.

While the freedoms granted by GPLv3 are respected in full, the project encourages:

- Collaboration over fragmentation
- Contribution over isolation
- Transparency over ownership disputes
- Stewardship over control
- Community benefit over individual gain

---

## Open Source Principles

Batocera Linux respects and embraces the freedoms provided by GPLv3 and other compatible open-source licenses.

Users and developers are free to:

- Use the software
- Study the software
- Modify the software
- Redistribute the software
- Create derivative works

These freedoms are fundamental to the project's success and long-term sustainability.

---

## Community Expectations

The Batocera Linux Project encourages all participants to:

- Respect contributors and maintainers
- Provide attribution where appropriate
- Share improvements upstream whenever practical
- Avoid unnecessary ecosystem fragmentation
- Act in good faith toward the project and community

These expectations are community principles rather than legal obligations.

---

## Forks and Derivative Projects

Forking is an important freedom provided by GPLv3 and has contributed significantly to the success of open-source software.

The Batocera Linux Project asks that this freedom be exercised responsibly.

Where practical, contributors are encouraged to collaborate with the upstream project so that improvements benefit the broader community rather than remaining isolated in long-lived independent forks.

Derivative projects should clearly distinguish themselves from official Batocera Linux releases.

See TRADEMARK.md and DISTRIBUTION.md.

---

## Official Releases

Only releases published through official Batocera Linux Project channels may be described as Official Batocera Linux Releases.

Modified builds, derivative distributions, and community-maintained forks must clearly identify themselves as unofficial.

---

## Respect for Contributors

Batocera Linux exists because individuals have voluntarily invested their time, expertise, and creativity into the project.

Users, redistributors, and derivative projects are encouraged to acknowledge this work and preserve contributor attribution wherever practical and legally appropriate.

The project strongly discourages any attempt to obscure, remove, or misrepresent the contributions of others.

---

## Governance Evolution

The Batocera Linux Project may update governance documents as the project evolves.

Changes should remain consistent with:

- GPLv3
- Open-source principles
- Community collaboration
- Long-term sustainability
